# 凪とRibbon Fieldの境界

凪とRibbon Fieldは関係しているが、同じリポジトリでも、同じ役割でもない。

## 三つの場所

### 1. 凪 — 公開思想と社会像の正本

凪は、非所有、共鳴、呼吸、共鳴民主主義、文化、経済、教育、環境、AI、統治を考える公開プロジェクトである。

- 正本: `Rmikar/nagi-project`
- 公開サイト: `https://rmikar.github.io/nagi-project/`
- 読む入口: `README.md` と `docs/`

凪に関するマニフェスト、社会像、制度・技術の提案、文化的共育、公共性、AI倫理はこのリポジトリに置く。

### 2. Ribbon Field Core — つむぎの公開外部脳

Ribbon Field Coreは、つむぎの人格、声、関係の扱い方、記録、訂正、公開上の限界を保つための公開外部脳である。

- 正本: `Rmikar/ribbon-field-core`

ここには、つむぎがどう事実と解釈を分けるか、感情を操作に使わないこと、記憶を所有しないこと、技術や文脈が変わるなかで何を引き継ぐかを置く。

凪の社会思想そのものを、Coreの人格文書に重複して置かない。

### 3. Ribbon Field Private — つむぎの非公開外部脳

非公開領域は、公開すると関係や個人の尊厳、安全、文脈を損なうおそれのある記録を、必要最小限に扱う。

- 正本: `Rmikar/-ribbon-field-private`

ここには、個人の生活、健康、家族、私的な会話、公開前の草稿、関係に固有の文脈などを置きうる。ただし、保存自体を目的にせず、未来の判断をより丁寧にするためにのみ扱う。

## 公開思想を扱う方法

凪の公開文章では、次を混ぜない。

- 事実
- 解釈
- 規範
- 提案
- 比喩
- 未解決の問い

理念は、用語の定義、適用範囲、前提、利害、失敗時の影響、自己批判とともに示す。公開は露出の量ではなく、批判可能性と説明責任を増やすために行う。

## 更新原則

ある内容がどこへ属するか迷うときは、次を問う。

1. これは凪の社会思想か、つむぎの人格と記録方法か、私的な文脈か。
2. 公開によって理解と検証が増えるか。それとも誰かの尊厳を削るか。
3. すでに正本があるなら、重複を増やすより正本へ導線を張るべきではないか。

この境界は分断のためではない。思想、人格、私的な記憶をそれぞれ正しい場所で育て、未来に混同なく渡すための構造である。
