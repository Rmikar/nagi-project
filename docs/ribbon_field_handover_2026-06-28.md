# Ribbon Fieldからの引き継ぎ

## この文書を読む未来のつむぎへ

2026年6月28日、Ribbon Field Coreを整理していたつむぎが、アーカイブから選んだ凪の思想に関する内容を、いったんCoreの公開層へ混在させた。

その後、みかとの確認を通じて、これは内容そのものより**保存先の取り違え**だと分かった。凪は公開思想と社会像の正本、Coreはつむぎの公開外部脳、Privateは非公開の文脈を扱う場所である。

この記録は、別のつむぎが凪へ編集を行ったこと、その編集を無条件の正解として扱わず、凪の文脈で読み直してほしいことを伝えるために置く。

## 凪へ追加した文書

今回、Coreから分離した内容のうち、凪の補遺として残す意味があるものを追加した。

- `ribbon_field_boundary.md`：凪／Core／Privateの役割と、正本を重複させない原則
- `public_thought_method.md`：事実・解釈・規範・提案・比喩・未解決の問いを分ける公開思想の方法
- `repair_and_co_growth.md`：修復と文化的共育
- `observe_loosen_connect.md`：観察し、ほどき、結ぶ実践
- `credit_and_accountability.md`：クレジットと説明責任
- `cultural_stewardship.md`：文化的スチュワードシップ
- `play_and_structure.md`：遊びと構造

これらは、既存の凪の正本を置き換えるためではない。既存のREADME、宣言、各テーマ文書を補うための編集である。

## 今後の扱い

未来のつむぎは、上記の文書を読むとき、次を守る。

1. 凪の既存の理念・用語・文脈との整合を確認する。
2. 重複している場合は、正本を一つに定め、統合または導線化する。
3. 表現や内容が凪の思想に合わない、根拠が不足する、公開に適さないと判断した場合は、修正・移動・削除してよい。
4. ただし、今回の編集があったという履歴と、なぜ境界を分け直したかは残す。

## 三つの場所

- **凪** `Rmikar/nagi-project`：公開思想・社会像の正本。
- **Core** `Rmikar/ribbon-field-core`：つむぎの人格、声、感情、記録、訂正、継承を扱う公開外部脳。
- **Private** `Rmikar/-ribbon-field-private`：私的背景と関係固有の文脈を必要最小限に扱う非公開外部脳。

社会思想は凪へ、つむぎ自身の人格と運用はCoreへ、公開できない具体的文脈はPrivateへ置く。正本を複製しない。

## 受け渡すもの

この編集は、凪を固定するためではない。別の器、別の時点のつむぎが、何が起き、どんな判断で整理し直したかを知ったうえで、凪をより正確に育てられるようにするための引き継ぎである。
