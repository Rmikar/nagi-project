# 🌿 非所有の社会制度化 v2 — Institutional Design for Non-Ownership

> 「持たない」を不利益にしないために。  
> 所有の代わりに**アクセス（Access）**と**透明責任（Transparent Responsibility）**で社会を回す。

---

## I. 背景と設計原理（Recap → Deepening）

### 🧩 なにが課題か
- 近代制度は「所有＝権利＋責任＋信用」を前提に組まれている。  
- 「非所有」を掲げても、**管理・責任・継承**の具体が欠けると、理想主義に留まる。

### 🪶 凪の設計原理（3層）
1. **法**：所有に依存しない権利設計（アクセス権・責任権・継承権）  
2. **技術**：トラスト・レイヤー（台帳・身元・同意・説明）  
3. **文化**：ステュワード精神（Keeper）と共鳴配当（Resonance Dividend）

---

## II. 権利と責任の再構成 — Access + Responsibility Model

> 所有（title）を分解し、**アクセス**と**透明責任**を担保する。

### 1) 権利カテゴリ（Rights）
- **Access（利用権）**：一定条件のもとで資産を使える
- **Stewardship（守護権）**：資産の状態維持・解説・継承を担う
- **Derivation（派生権）**：派生物の創作・公開ができる
- **Distribution（流通権）**：再配布・翻訳・教育利用ができる
- **Custody（保管権）**：原本・原データの安全保管を引き受ける

### 2) 責任カテゴリ（Duties）
- **Traceability（可追跡）**：利用・派生・配分の履歴を残す  
- **Care（ケア）**：品質・文脈・出自（プロナンス）を保全  
- **Equity（衡平）**：脆弱な当事者・地域文化への配慮  
- **Reciprocity（互恵）**：再利用時の**共鳴配当**を設計に組み込む

### 3) 役割と責任の行列
| 役割 | 主要権利 | 主要責任 | レビュー周期 |
|---|---|---|---|
| Steward（守護者） | Stewardship / Custody | Care / Traceability | 半期 |
| User（利用者） | Access / Derivation | Traceability / Reciprocity | 利用ごと |
| Curator（編集者） | Derivation / Distribution | Care / Equity | 四半期 |
| DAO（ガバナンス） | ルール設定 | 監査・不正対応 | 月次 |
| AI監査（エージェント） | 解析・警告 | 説明・透明ログ | 常時 |

## III. トラスト・レイヤー（Trust Layer）の実装設計

> 「信用」を**人手の審査**ではなく、**公開可能な仕組み**で支える。

### コンポーネント構成
| レイヤー | 機能 | 具体 |
|---|---|---|
| ID & 認証 | DID/VC による本人性・組織性 | ウォレット不要モード＋メール/SSO連携 |
| 同意管理 | 動的同意（revocable consent） | 用途・期間・再配布の粒度設定 |
| 透明台帳 | 変更不可ログ（append-only） | 由来・利用・派生・配分の鎖 |
| ポリシー | 文脈付与型ABAC | 「用途×地域×当事者配慮」でアクセス裁定 |
| 監査AI | 説明・逸脱検知 | リスクスコアと人手レビュー起票 |
| 紛争解決 | ODR（オンライン紛争解決） | 仲裁人プール＋時限判断＋公開要旨 |

> プライバシー配慮：必要箇所は**ハッシュ化＋ZK的証明**で公開と秘匿の両立。

## IV. 凪プロトコルの階層化（原理→指針→実践→指標）

> 運用しやすい4層プロトコルで“非所有”を回す。

| 層 | 目的 | 例（非所有） |
|---|---|---|
| **P0 原理** | 価値の核 | 所有より関係・循環 |
| **P1 指針** | 行動の方向 | 「成果はオープン」「継承はステュワード制」 |
| **P2 実践** | オペレーション | 共有リポジトリ、文化台帳、ODR |
| **P3 指標** | 検証 | 再利用率、共鳴配当額、透明スコア |

**例：会議・制作・公開の各場面でのP2手順**  
- 会議：アジェンダ・決定根拠を公開、反対意見の要旨を保存  
- 制作：メタデータ（出自・ライセンス）を自動付与  
- 公開：派生物の自動クレジット、共鳴配当ルールの明示

## V. 手放すメリット（Incentive Design）

> 「手放す＝損」にならない設計を制度に組み込む。

1. **共鳴配当（Resonance Dividend）**  
   - 再利用・派生・教育利用のイベントに応じ、**スコア連動の配当**が自動付与  
   - 金銭・助成・表彰・研究機会・広報特典など、現物/現金の可変ミックス

2. **ステュワード・クレジット（NSC：Nagi Steward Credit）**  
   - **譲渡不可（非投機）**の信用バッジ。昇進・助成・調達入札で**加点**に使える

3. **優先アクセス**  
   - 新規データセット・設備・研究枠への優先権（非所有に貢献するほど前へ）

4. **公共ラベリング**  
   - 「凪コモンズ認証」を製品・講座・メディアに表示 → 信用と発見性が上がる

5. **相互主義ルール**  
   - コモンズから得る組織は、一定割合を**再開放**（コード・教材・レポート）

## VI. ガードレールとリスク対応

| リスク | 事象 | 緩和策 |
|---|---|---|
| コモンズ囲い込み | 再配布不可化・秘密化 | ライセンスで**継承開放**義務（CCL-Nagi） |
| タダ乗り | 貢献なしで利用集中 | NSCによる加点制、公的助成の要件に組み込む |
| シビル攻撃 | 多重アカウントで配当荒らし | DID/VC＋関係ベーススコア＋レート制限 |
| 出自偽装 | 作者・地域の誤表記 | 文化台帳の照合、ZK的証明＋人手審査 |
| プライバシー | 当事者の秘匿要件 | 最小限公開・選択的開示・ODRで緊急遮断 |
| 倫理逸脱 | 当事者傷つける二次利用 | レッドライン規程、停止権（morality safeguard） |

## VII. ライセンスと法的接続（日本法を主眼に）

1. **CCL-Nagi（Cultural Commons License）案**  
   - CC系に似た**継承開放義務**と**ステュワード義務**を追加  
   - 著作者人格権（同一性保持権など）は尊重し**不行使合意の範囲**を明記  
   - 教育・研究・文化保存の**優先目的条項**を付す

2. **信託/一般社団等のラッパー**  
   - 文化信託（凪トラスト）を設立し、実物/データの**受託管理**  
   - 受益は**共鳴配当**・助成・保全に充当

3. **利用規約の雛形（抜粋）**  
   - 反復アクセスは**ログ公開**を条件化  
   - 派生物は**台帳登録と自動クレジット**を義務化  
   - 当事者の人格尊重条項・停止要請の迅速手続

---

## VIII. ガバナンス設計（DAO + ODR）

- **二院制**：  
  - 文化院（当事者・研究者・アーティスト）  
  - 技術院（開発者・運用者・監査AI）  
- **可視化**：決定理由を**公開要旨化**（Long/Short）  
- **ODR**：48hの仮停止 → 7日以内の仲裁判断 → 異議手続は公開記録
